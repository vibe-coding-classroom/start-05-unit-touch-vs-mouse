針對 **`start-05-unit-touch-vs-mouse` (觸控 vs. 滑鼠：打造全平台通用的操控體驗)** 單元，這是學員建立「跨裝置相容性 (Cross-Device Compatibility)」認知的實戰課。它要求學員具備 **事件序列管理 (Event Sequencing)** 與 **輸入正規化 (Input Normalization)** 的能力，學習如何處理觸控特有的 300ms 延遲、攔截「幽靈點擊 (Ghost Clicks)」，並實作一個能同時應對滑鼠與指尖的萬用操控介面。

以下是在 **GitHub Classroom** 部署其作業 (Assignments) 的具體建議：

### 1. 範本倉庫 (Template Repo) 配置建議
跨裝置作業的核心在於「事件攔截與統一座標解析」，範本應包含多種輸入情境的測試面板，建議包含：
*   **📂 `index.html`**：**萬用測試面板**。預設包含一個大按鈕與一個滑動條，並內建「事件監控日誌」，讓學員能清楚看到在不同裝置（桌面 vs 行動端）下，事件發生的順序（如 `touchstart` -> `mousedown` -> `click`）。
*   **📂 `touch-logic.js`**：**事件攔截實作區**。學員在此實作「萬用座標解析器 (Universal Position Helper)」。預留 `// TODO` 引導學員撰寫邏輯來分辨 `e.clientX` 與 `e.touches[0].clientX`。
*   **📂 `README.md`**：**相容性測試報告 (Compatibility Log)**。提供模板要求學員錄製或紀錄：在沒加入 `preventDefault()` 之前，行動端點擊一次按鈕會觸發幾次 Action？延遲大約是多少毫秒？
*   **📂 `assets/`**：**事件觸發順序流程圖**。標示從 `touchstart` 到 `click` 的完整鍊條，輔助學員理解攔截的最佳切入點。

### 2. 作業任務部署細節

#### 任務 1：打造全平台萬用按鈕 (Omni-Button Implementation Lab)
*   **目標**：掌握「事件競態條件 (Race Condition)」。同時監聽 `mousedown` 與 `touchstart`，但確保在行動端僅觸發一次執行邏輯，徹底消除 300ms 延遲。
*   **Classroom 部署建議**：
    *   **幽靈點擊攔截實作**：在 `touchstart` 事件中使用 `e.preventDefault()` 斬斷後續模擬的 `mousedown` 鍊條。
    *   **異步處理挑戰**：確保按鈕在快速連續點擊下，依然維持高度的穩定性。
    *   **驗證要點**：**唯一性與速度感審核**。在手機上點擊時，後台日誌是否僅出現一筆 Action？延遲是否感覺「歸零」？當切換回電腦滑鼠時，功能是否依然正常？

#### 任務 2：防抖滑動控制 (Interaction Locking Lab)
*   **目標**：掌握「被動監聽器 (Passive Listener) 攔截術」。製作一個橫向滑動條，確保使用者在拖曳時，行動端網頁不會發生「意外的上下捲動」。
*   **Classroom 部署建議**：
    *   **視窗滾動鎖定實作**：對滑桿容器設定 `{ passive: false }`，並在 `touchmove` 中精確執行 `preventDefault()`。
    *   **統一座標輔助函數**：實作一個 `getPosition(e)` 函數，能自動適應滑鼠物件或觸控物件的座標提取路徑。
    *   **驗證要點**：**操作流暢度審核**。拖曳滑桿時，背景頁面是否完全靜止？手指快速滑動是否有位移或跳動感？

#### 任務 3：原生 App 體感優化 (Native Feel UX Lab)
*   **目標**：掌握「瀏覽器行為防禦」。封鎖長按時的系統右鍵選單 (Context Menu) 與預設的點擊高亮色，讓 Web App 質感升級。
*   **Classroom 部署建議**：
    *   **系統選單屏蔽實作**：監聽 `contextmenu` 事件並回傳 `false` (或執行 `preventDefault`)，消除所有阻礙操控的對話框。
    *   **視覺質感清潔實作**：使用 CSS 清除行動端預設的藍色 Tap 框，並防止在激烈操作中誤選取文字。
    *   **驗證要點**：**APP 質感模擬審核**。長按按鈕是否不再跳出「複製/儲存圖片」？按鈕按下時是否有自定義的物理回饋（如震動感或 CSS 陰影變化）？

### 3. 跨平台邏輯健壯性與 UX 潔淨度點評標準 (Hybrid Input Standards)
此單元的價值在於 **「對底層事件生成的理解深度與對跨平台差異的適配精準度」**：
*   [ ] **事件防重處理精確度**：是否能正確使用攔截機制防止雙重觸發？
*   [ ] **座標解析通用性**：封裝的輔助函數是否能無縫處理所有主流瀏覽器的滑鼠與觸控物件？
*   [ ] **介面穩定性**：是否成功消除了所有會中斷硬體控制的瀏覽器預設行為（如捲動、縮放、選單）？
